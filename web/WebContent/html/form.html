입력 양식

form : 사용자의 입력을 받는 화면을 구성할 때 사용


입력한 데이터를 다른 페이지로 보낼 수 있음
ex) 로그인 할 때 아이디와 비밀번호를 입력하는 경우
회원가입 시 필요한 여러 정보를 입력하는 경우

form 태그를 통해 입력 데이터를 내보내려면 name 태그 필요
form 태그의 action 속성값이 없거나 action을 지정하지 않으면 자기 자신의 페이지 연결
autocomplete : 입력 태그의 자동완성 기능으로 기본은 on인 상태 사용하지 않을 시 off

파라미터 : 전송되는 입력 값
파라미터가 여러 개인 경우 $로 구분
파라미터가 노출 되는 형태 : method : get
파라미터가 노출 되지 않는 형태 : method : post

placeholder : 텍스트 입력 태그에 무엇을 입력하는 태그인지 보여줌
required : 필수 입력 항목을 입력하지 않을 시 submit 되지 않도록 처리하려면 프로그래밍적으로 해결할 수도 있지만 태그 자체의 required 속성을 이용할 수 도 있다

readonly : 입력 태그이지만 변경 불가하게 처리
disabled : 입력 태그이지만 변경 불가하게 처리
차이점 : readonly 속성을 갖는 태그는 submit 시 데이터가 전송
disabled 속성을 갖는 태그는 submit 시 데이터가 전송되지 않음

파일을 첨부하여 전송하는 경우는 반드시 method를 post로 지정해야만 한다
enctype을 multipart/form-data로 지정해야만 한다

input

text 한 줄 텍스트 입력
password 입력 데이터가 보이지 않는 태그
radio 여러 항목 중에 한 개만 선택할 때 사용
hidden 실제 소스에는 있지만 화면에서는 보이지 않는 태그
checkbox 여러 항목을 모두 선택할 수 있는 태그
file 첨부할 파일을 선택하는 태그
submit 입력 데이터를 전송하는 버튼


textarea 여러 줄을 입력하는데 사용하는 입력 텍스트 태그
select 여러 개의 목록에서 선택하는 드랍다운 목록 태그
